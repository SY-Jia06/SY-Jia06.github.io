# 文章页三栏重设计

日期：2026-03-29

## 目标

把当前文章页从“单栏正文 + 局部阅读增强”升级为更完整的阅读场景：

- 左栏：当前文章目录
- 中栏：正文与评论区
- 右栏：同分类文章

目标不是把博客做成文档站，而是在保持现有站点气质的前提下，让长文阅读更舒服、更专业、更有结构感。

## 设计结论

本轮采用中度改版，而不是只做局部增强。

页面信息架构固定为：

- 左栏放目录
- 中栏放正文
- 右栏放同分类文章

两侧栏都不固定，不做吸附；跟页面一起滚动。

移动端不保留三栏，改为折叠回正文上下结构。

## 为什么这样定

### 目录放左边

因为当前阅读习惯下，左侧更适合承接“先建立结构感”的信息。目录是当前文章最强相关的辅助信息，优先级高于推荐内容，放左侧更顺。

### 同类文章放右边

同类文章是阅读延伸，不是当前阅读主线。它应该存在，但不应该抢正文注意力，所以更适合放在右边作为“次级浏览入口”。

### 两侧不固定

当前博客的气质偏安静、克制，不适合做成工具型页面。固定侧栏虽然提高功能性，但会明显增加“文档站”感，也会让页面更重。非固定侧栏更自然，更像博客。

## 页面结构

桌面端改为三栏阅读布局：

```html
<div class="article-layout article-layout-reading">
  <aside class="article-side article-side-left">
    <!-- TOC -->
  </aside>

  <main class="article-main">
    <!-- hero -->
    <!-- post shell -->
    <!-- post pagination -->
    <!-- comments -->
  </main>

  <aside class="article-side article-side-right">
    <!-- related posts in same category -->
  </aside>
</div>
```

其中：

- 左栏宽度窄，强调结构
- 中栏占绝对主宽
- 右栏略窄于左栏或相近，强调“继续阅读”

视觉权重必须明确偏向中栏。

## 中栏设计

中栏仍然是主角，继续沿用当前站的审美方向：

- 保留现有 hero 和正文主卡片
- 保留评论区在正文之后
- 保留阅读进度条
- 保留上一篇 / 下一篇导航

但会调整正文场的整体比例和边距，使它在三栏结构下依然有足够呼吸感。

核心要求：

- 正文宽度不能被压得太窄
- 评论区仍然属于正文阅读流的一部分
- 上下篇导航放在正文底部、评论区之前

## 左栏设计：目录

左栏只放目录，不混入其他信息。

目录行为：

- 继续只抓 `h2` / `h3`
- 继续沿用阈值：至少 `1 个 h2 + 1 个 h3`
- 当前阅读位置对应项高亮
- 点击目录项平滑滚动

目录视觉：

- 不是卡片堆叠，而是更像一块清爽的章节导航面板
- 标题层级清晰，但不能比正文标题更强
- 当前项高亮明显，但整体保持克制

## 右栏设计：同分类文章

右栏展示当前文章所在分类下的其他文章，不展示跨分类推荐。

原因：

- 当前数据结构里，分类是最稳定的聚合维度
- 这比标签相关推荐更容易保证相关性
- 先做同分类文章，后面若要升级为“系列文章”也更容易演进

展示规则：

- 排除当前文章
- 默认按现有文章排序规则展示
- 数量控制在较小范围内，避免侧栏过长
- 标明分类语义，例如“同分类文章”

## 移动端策略

移动端不保留左右三栏。

顺序改为：

1. hero
2. 目录
3. 正文
4. 上下篇
5. 同分类文章
6. 评论区

原因：

- 小屏强行三栏没有意义
- 目录仍保留较高优先级
- 同分类文章放在正文之后更自然

## 与现有实现的关系

当前刚做好的 `reading-enhancements.js` 不废弃，而是继续作为阅读增强核心模块存在。

调整方式：

- 保留阅读进度逻辑
- 保留目录逻辑，但目录从正文内部位置移到左栏
- 保留上一篇 / 下一篇
- 新增“同分类文章”逻辑，可放在同一模块中，或拆为阅读侧栏相关逻辑

`blog-app.js` 仍然只负责：

- 加载文章
- 渲染正文
- 挂载评论
- 触发阅读场布局渲染

## 样式方向

参考对象只借布局思路，不借视觉皮肤。

保留你当前站点的方向：

- 浅色、留白、圆角、干净卡片
- 不切成深色赛博风
- 不做重玻璃、重阴影、重动效

新布局需要体现的是：

- 结构更完整
- 信息层级更清楚
- 正文聚焦感更强

而不是“像别人那个站”。

## 风险

- 如果三栏比例没控好，正文会被挤窄
- 如果左右栏视觉太重，会让页面失去博客感
- 如果右栏同分类文章过多，会破坏页面重心
- 如果移动端回落策略处理不好，会出现信息顺序混乱

## 暂不做

- 固定侧栏
- 标签相关推荐
- 跨分类推荐
- 右侧悬浮目录
- 更换整体主题风格
- 首页 / 列表页联动重设计

## 验收标准

- 桌面端文章页呈现稳定三栏结构
- 左栏目录清晰可用
- 中栏正文仍然最聚焦
- 右栏同分类文章不喧宾夺主
- 移动端自动回落为单栏顺序布局
- 评论区仍正常工作
- 阅读进度、目录高亮、上下篇导航都不回退
