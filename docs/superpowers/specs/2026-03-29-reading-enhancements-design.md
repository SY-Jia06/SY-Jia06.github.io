# 阅读增强设计

日期：2026-03-29

## 目标

在不引入新依赖、不改动现有单页博客路由模型的前提下，提升单篇文章的阅读体验，让文章页更专业、更舒服。

本轮只做 3 个能力：

- 阅读进度条
- 单篇文内目录
- 上一篇 / 下一篇导航

## 现状

当前博客是静态单页结构：

- 文章列表和筛选在 `blog.html`
- 单篇文章通过 `blog-app.js` 动态读取 Markdown 并渲染到 `#postContent`
- 文章切换依赖 `?post=<post-id>`
- `POSTS` 数据来自 `posts.js`

现有结构已经支持：

- 列表浏览
- 搜索 / 标签 / 分类筛选
- 单篇阅读
- Giscus 评论区

缺少的是单篇阅读期的“定位感”和“继续阅读路径”。

## 设计原则

- 不新增第三方依赖
- 不把现有博客改造成多页面路由
- 不继续把 `blog-app.js` 变成阅读功能大杂烩
- 阅读增强能力集中到独立模块
- 默认兼容桌面和移动端
- 对短文章保持克制，不因为增强而制造噪音

## 模块边界

新增模块：

- `reading-enhancements.js`

职责：

- 从渲染后的文章正文中提取 `h2` / `h3`
- 生成并更新文内目录
- 监听滚动并更新阅读进度
- 依据当前文章在 `POSTS` 中的位置生成上一篇 / 下一篇
- 在文章关闭或切换时清理自身状态

不负责：

- Markdown 拉取
- Markdown 渲染
- 评论区挂载
- 文章 hero / meta 渲染

这些继续由 `blog-app.js` 负责。

## 页面结构

在单篇阅读区新增 3 个区域：

1. 顶部细阅读进度条
2. 正文开头目录卡片
3. 正文底部上一篇 / 下一篇导航

推荐结构：

```html
<div class="reading-progress" id="readingProgress" hidden>
  <span class="reading-progress-bar" id="readingProgressBar"></span>
</div>

<article class="post-shell post-shell-reading">
  <aside class="post-toc" id="postToc" hidden></aside>
  <article class="post-content" id="postContent"></article>
  <nav class="post-pagination" id="postPagination" hidden></nav>
</article>
```

## 功能设计

### 1. 阅读进度条

行为：

- 打开文章后显示在页面顶部
- 进度按正文阅读区块计算，不按整个页面高度粗算
- 阅读结束时进度接近 100%
- 返回列表后隐藏

计算口径：

- 起点：正文内容区进入视口顶部
- 终点：正文内容区滚动到底部

这样比按整页滚动更准确，也不会被 hero、评论区或页脚干扰。

### 2. 单篇文内目录

行为：

- 从 `#postContent` 中抓取 `h2` / `h3`
- 为目录项生成锚点
- 点击目录项平滑跳转
- 当前阅读位置对应的标题项高亮

显示策略：

- 只有当文章存在足够标题时才显示
- 默认阈值：至少 2 个 `h2`，或 1 个 `h2` + 1 个 `h3`
- 标题过少则自动隐藏目录卡片

布局策略：

- 使用“正文开头目录卡片”
- 不做右侧悬浮目录

原因：

- 当前页面是单列阅读布局，正文开头卡片更稳
- 移动端无需额外折叠或吸附策略
- 实现成本低，视觉上更克制

### 3. 上一篇 / 下一篇

行为：

- 在文章正文底部显示导航卡片
- 根据当前 `POSTS` 的排序结果确定上一篇 / 下一篇
- 第一篇只显示下一篇
- 最后一篇只显示上一篇

排序来源：

- 以现有 `sortPostsByDate()` 为准

这样可以与列表页排序保持一致，避免“列表顺序”和“文章顺序”不一致。

## 交互流程

1. 用户打开文章
2. `blog-app.js` 完成 Markdown 渲染
3. `blog-app.js` 调用 `reading-enhancements.js` 的初始化方法
4. 模块生成目录、上下篇、进度监听
5. 用户滚动时更新：
   - 顶部进度条
   - 目录高亮项
6. 用户切换到另一篇文章或返回列表时：
   - 清理滚动监听
   - 清空目录和分页内容
   - 隐藏进度条

## 接口草案

建议对外只暴露 2 个方法：

```js
window.ReadingEnhancements = {
  mount({ post, posts, contentElement }),
  unmount()
};
```

参数说明：

- `post`: 当前文章对象
- `posts`: 全部文章数据
- `contentElement`: `#postContent`

## 样式策略

新增样式应延续当前语言：

- 继续使用现有圆角、边框、留白和阴影系统
- 不引入重悬浮、重玻璃态、过强吸附动画
- 目录卡片视觉层级低于 hero，高于正文
- 上下篇导航应像“继续阅读卡片”，不是普通文本链接

移动端策略：

- 目录卡片保持块级堆叠
- 上下篇导航改为纵向排列
- 顶部进度条保留

## 失败与边界处理

- 没有正文标题：隐藏目录
- 标题重复：生成去重锚点
- 标题中有标点或中文：锚点需安全转义
- 文章极短：进度条仍可工作，但目录和分页按条件显示
- 切换文章多次：必须避免重复注册滚动监听

## 测试策略

最少验证应包括：

- 打开长文后目录正常出现
- 点击目录项可跳到对应段落
- 滚动时目录高亮变化正常
- 滚动时进度条变化正常
- 第一篇 / 最后一篇文章分页显示正确
- 切换不同文章后目录与分页同步刷新
- 返回列表后增强 UI 消失
- 移动端布局未被破坏

## 风险

- `blog-app.js` 与新模块的挂载时机若处理不好，可能出现目录抓取为空
- 若锚点生成规则不稳定，目录跳转和高亮会错位
- 若滚动监听未清理，切换文章后可能出现重复更新

## 暂不做

- 右侧悬浮目录
- 阅读时长
- 图片点击放大
- 代码块复制
- 自动高亮当前段落

这些都属于后续阅读增强候选项，但不进入本轮范围。

## 推荐实施顺序

1. 建立 `reading-enhancements.js` 模块骨架
2. 接入上一篇 / 下一篇
3. 接入目录生成与跳转
4. 接入阅读进度条
5. 调整样式与移动端布局
6. 完成回归验证
